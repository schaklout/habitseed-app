## Summary

A mobile-first habit tracking application that allows users to create habits, mark daily completions, and view progress. The app prioritizes local-first operation with optional cloud synchronization via a secure REST API backed by MySQL. React Native enables cross-platform iOS/Android deployment with a single codebase.

## Technical Context

**Language/Version**: JavaScript/TypeScript (React Native 0.72+)  
**Primary Dependencies**: React Native, Expo (or bare workflow), Redux or Zustand for state, SQLite for local storage, Axios or Fetch for HTTP  
**Storage**: AsyncStorage (encrypted locally on device) + MySQL (remote, optional cloud sync)  
**Testing**: Jest (unit/integration), Detox (UI/e2e)  
**Target Platform**: iOS 15+, Android 12+  
**Project Type**: Mobile app  
**Performance Goals**: App startup <3 seconds, habit list load <2 seconds, completion tap response <500ms, offline operation seamless  
**Constraints**: <100MB install size, encrypted sensitive data at rest and in transit, <50ms UI frame drops, works offline without network  
**Scale/Scope**: Single user per device, up to 100+ habits, complete history synced to cloud

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

✅ **Simple Architecture**: Monolithic mobile app (React Native) with single REST API backend (Node.js/Express) and MySQL. No microservices or over-engineering.

✅ **Readable Code**: Will enforce with ESLint, Prettier, TypeScript for type safety and clarity. Code reviews will focus on naming and simplicity.

✅ **Minimal Dependencies**: React Native chosen for cross-platform single codebase. SQLite (native) and AsyncStorage for local storage. MySQL justified for secure remote persistence. Dependencies reviewed before integration.

✅ **Mobile-First Design**: Entire UI built with React Native components, optimized for touch and small screens. Offline operation preserved (local-first with optional cloud sync).

✅ **Local-First Data**: AsyncStorage with encryption provides fast local access. MySQL sync is optional/background, not blocking. Users can work offline entirely.

✅ **Security & Privacy**: 
- AsyncStorage encrypted locally using react-native-keychain or crypto library
- MySQL credentials via .env (never in repo)
- API uses HTTPS only
- JWT tokens for authentication
- All sensitive data encrypted at rest and in transit
- User inputs validated server-side
